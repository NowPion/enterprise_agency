# 企业代办小程序需求分析文档

## 一、项目背景与目标

随着中小企业与个体工商户数量持续增长，工商年报、异常解除、注销等事务办理频率高、规则复杂、时间敏感。大量企业主缺乏专业知识，线下办理成本高、周期长，容易出现逾期或操作错误。

本项目拟建设一套 **企业代办小程序 + 后台管理系统**，通过线上下单、支付与进度跟踪，由平台提供标准化、可追溯的工商事务代办服务，降低企业办理门槛，提高办理效率。

------

## 二、用户角色分析

### 1. 小程序端用户

- 个体工商户
- 小微企业法人
- 财务/代账相关从业人员

**核心需求：**

- 快速了解政策与办理要求
- 便捷下单并完成支付
- 实时查看办理进度
- 留存办理材料与回执

### 2. 后台管理用户

- 超级管理员：系统配置、权限管理
- 运营人员：订单审核、派单、内容维护
- 办理人员：具体业务办理、材料上传
- 财务人员：支付对账、退款、发票管理

------

## 三、系统总体架构

系统分为两大部分：

- **小程序端**：面向用户，负责业务下单、支付、进度查询与内容展示
- **后台管理系统**：面向内部人员，负责订单全生命周期管理、内容配置、支付与数据统计

系统采用前后端分离架构，核心业务数据统一由后台管理系统维护，并实时同步至小程序端。

------

## 四、小程序端功能需求分析

### 1. 核心业务办理

为用户提供营业执照相关代办服务，包括年报补报、异常解除、注册、注销等。

**功能要点：**

- 用户填写企业及法人信息
- 选择办理业务类型
- 勾选服务协议后提交申请
- 信息校验通过后进入后台审核流程

**风险与约束：**

- 办理成功后不支持退款
- 企业与法人信息需进行合法性校验
- 身份证号、手机号等敏感信息需加密或脱敏处理

------

### 2. 业务进度展示

进度展示分为两部分：

1. 示例进度：展示其他用户加急办理的示例数据，用于引导用户决策
2. 实际进度：展示用户自身订单的真实办理进度

**设计说明：**

- 示例数据仅用于展示，不参与真实业务计算
- 实际进度与后台订单节点实时同步

------

### 3. 常见问题答疑

按业务类型（年报 / 注销 / 解异常）分类展示常见问题，减少人工咨询压力。

**内容来源：**

- 由后台 CMS 模块统一维护
- 支持官方渠道链接与政策说明

------

### 4. 办理说明展示

向用户展示：

- 工商年报时间要求
- 逾期后果说明
- 平台代办服务优势

说明内容需支持后台实时更新，以适应政策变化。

------

### 5. 订单管理

用户可查看自身所有订单，包括：

- 待付款
- 办理中
- 已完成
- 售后状态

支持查看订单详情、进度节点及办理材料回执。

------

### 6. 个人中心

个人中心提供以下功能入口：

- 订单列表
- 优惠券查看
- 业务办理入口
- 问题反馈

用户需绑定手机号后方可使用完整功能。

## 五、后台管理系统功能需求分析

### 1. 账号与权限管理

系统后台采用 **基于角色的访问控制（Role-Based Access Control，RBAC）** 权限模型，对不同岗位人员的系统操作权限进行统一管理与隔离控制。

RBAC 模型的核心思想是：**用户不直接拥有权限，而是通过被分配的角色间接获得权限**，从而实现权限配置的集中化、标准化与可维护性。

------

#### （1）RBAC 模型结构说明

系统权限体系由以下三部分组成：

- **用户（User）**：后台登录账号，对应具体工作人员
- **角色（Role）**：岗位身份的抽象，如超管、运营、财务等
- **权限（Permission）**：系统中可执行的最小操作单元，如查看订单、退款、配置系统参数等

三者关系如下：

```text
用户（User） → 角色（Role） → 权限（Permission）
```

------

#### （2）角色划分设计

根据企业代办业务实际运行需求，系统设计以下核心角色：

- **超级管理员**：负责系统级配置与权限管理，拥有最高权限
- **运营人员**：负责订单审核、派单、进度管理及内容维护
- **办理人员**：负责具体工商事务的实际办理与材料上传
- **财务人员**：负责交易对账、退款处理与发票管理

------

#### （3）权限粒度设计

系统将权限拆分为多个原子级操作，避免使用“全模块权限”带来的安全隐患。典型权限示例如下：

- `order:view`：查看订单
- `order:review`：审核订单
- `order:assign`：派单
- `order:update`：更新办理进度
- `order:refund`：订单退款
- `cms:edit`：内容编辑
- `finance:view`：查看财务数据
- `system:config`：系统参数配置

------

#### （4）角色与权限对应关系示例

| 角色       | 权限说明                                     |
| ---------- | -------------------------------------------- |
| 超级管理员 | 拥有系统全部权限                             |
| 运营人员   | 查看订单、审核订单、派单、更新进度、内容管理 |
| 办理人员   | 查看订单、更新办理进度、上传办理材料         |
| 财务人员   | 查看订单、退款操作、财务数据与发票管理       |

------

#### （5）权限控制与安全审计

- 所有敏感操作（如退款、系统配置）均需权限校验
- 系统记录完整的登录日志与操作日志
- 操作日志可追溯至具体账号、角色及操作时间

通过 RBAC 权限设计，系统实现了 **职责分离、权限最小化、操作可追溯** 的安全管理目标。

------

### 2. 用户管理

统一管理小程序端用户信息，包括：

- 注册信息
- 手机号绑定状态
- 用户标签
- 黑名单管理

所有手机号需合规存储并在展示时进行脱敏处理。

------

### 3. 订单管理（核心模块）

覆盖订单全生命周期管理：

- 新建
- 审核
- 派单
- 办理中
- 完成
- 取消
- 售后

支持上传、回传执照、年报截图等办理材料，并与小程序端实时同步进度。

------

### 4. 业务产品管理

用于配置小程序端可展示的代办服务项目，包括：

- 服务定价
- 套餐配置
- 办理须知
- 表单字段定义
- 上下架控制

所有配置变更需实时同步至小程序端。

------

### 5. 支付与交易管理

系统对接微信支付渠道，实现：

- 交易记录管理
- 对账处理
- 退款与售后管理
- 发票申请与开具

实行交易对账日清制度，确保资金安全。

------

### 6. 内容管理（CMS）

统一管理小程序端内容，包括：

- 首页轮播图
- 公告
- 常见问题
- 办理说明

支持内容审核、版本回溯与上下架控制。

------

### 7. 数据统计与看板

统计并分析核心业务指标：

- DAU / MAU
- 订单量
- 转化率
- 客单价
- 用户留存

支持多维度筛选与报表导出。
